# ODL Interactive Process Guide (faculty-facing)

A self-contained web guide that shows faculty **(1)** the ODL project process,
**(2)** what's expected of *them* at each phase, and **(3)** a grounded estimate
of timeline + ODL effort + their own time.

## How to use it
- **Open it:** double-click `index.html` (works in any browser, no install, no internet needed).
- **Keep the two files together:** `index.html` loads `estimator.js` from the same folder.
- Pick a project **type** and **size** → the plan appears live. Use **"Fine-tune the
  deliverables"** to adjust exact counts (videos, animations, infographics, modules…).
- Tabs: **Your Plan** (estimates + per-phase detail), **The Process** (all phases), **Similar Projects**.
- **Print / save** button produces a clean PDF a faculty member can keep.

## Where the numbers come from (provenance)
- **CALIBRATED from ODL's logged time** (see `../CALIBRATION.md`): the `CALIBRATION`
  block in `estimator.js` is machine-generated by `../inject_calibration.py` — never
  edit it by hand. It carries archetype effort quartiles, the PM rate
  (0.6–1.6 h per calendar week), real timelines (full courses: median ≈ 31 weeks),
  and the validated video rate (~11–13 h per finished short video).
- **Media & graphics unit rates** = ODL's own per-deliverable estimation table
  (e.g., studio lecture ≈ 8.5h with a 4/0.5/4 Pre/Prod/Post split; explainer animation ≈ 100h).
  Edit these in `estimator.js` → `videoRates` / `graphicsRates` / `xrRates`.
  Lecture-band rates are validated by logged data; interview/mini-doc/XR are not yet.
- **Faculty-time figures (and Discovery/QA/Evaluation)** are ODL *planning estimates*
  (labeled in-app) — zero faculty hours are logged, so they cannot be calibrated yet.
- All figures are shown as **ranges**, never single numbers. The architecture keeps numbers
  deterministic (no AI invents a number), and `../refresh.py` re-calibrates everything
  as new time entries accumulate.

## To customize
- **Deliverable bundles per type×size:** `estimator.js` → `BUNDLES`.
- **Phase content (what happens / ODL does / you do / artifacts):** `estimator.js` → `PHASES`.
- **Showcase projects:** `estimator.js` → `SHOWCASE`.
